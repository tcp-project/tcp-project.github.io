What is Internet Protocol (IP)?

Internet protocol (IP) is the principle communication protocol used to relay datagrams across network boundaries. This protocol utilizes a routing function that allows inter-networking (connecting different networks to build a large, global network) which is what essentially establishes the Internet. IP works by delivering packets from source hosts to destination hosts based on the IP addresses found in the packet headers (a unique address that computing devices use to identity itself).

What is Transmission Control Protocol (TCP)?

Transmission Control Protocol (TCP) is one the main protocols of the Internet protocol suite. TCP was originally implemented to complement the Internet Protocl (IP) and was commonly referred to as TCP/IP. The primary application of the TCP communication protocol is to provide the most reliable and error-checked bi-directional stream of data between hosts on an IP network. Some examples where TCP communication can be found include email, the world wide web, file transfer, etc...

Connection Inititation

TCP uses a process known as a three-way handshake to establish a connection between the client and the server. This process works by the server listening at a specific port (i.e. port: 9600) and opens up the port for connections. A client may then initiate an active open at the port and proceed the three-way handshake to establish a connection with the server.

Connection Termination

When ending the connection between the sender and receiver a four-way handshake method is used instead of the three-way handshake. When one endpoint wants to stop its half of the data transmission it transmits a FIN packet which the other end acknowledges with an Ack. The typical connection termination involves a pair of FIN and Ack packets from both ends. It is also possible for connections to be "half-open" in which one side has terminated their connection but can continue to either read or send data.

Reliable Transmission

TCP uses sequence numbers to identify each byte of data that is sent from the client to the server by adding a unique sequence number to the TCP header containing the data. The sequence numbers associated with each byte of data sent are used by the receiver/destination to order the bytes of data received so that the data can be reconstructed in perfect order regardless of any possible packet loss or packet reordering that occurs during transmission. Acknowledgment numbers (Acks) are coupled with the sequence numbers so the receiver of the data can tell the sender whether the data/packet has been received. There are two main techniques in TCP utilizing these sequence numbers and Acks known as DupAcks and RTO.

Dupack-based re-transmission (DupAcks)

Lets say a single packet (packet 100) is lost in the data transmission stream. Then, the receiver can not acknowledge packets above 100 as it sends cumulative Acks back to the sender. This causes the receiver to acknowledge packet 99 again on the receipt of the next packet in the stream. This duplicate acknowledgment is used to signal a packet loss which prompts the sender to re-transmit the unacknowledged packet.

Timeout-based retransmission (RTO)

When a packet is sent by the sender, the sender sets a time limit (RTO) that is an estimate of how long it should take for the receiver to send back an Ack. If the time limit expires during data transmission the sender will re-transmit the packet. In some cases if the timer has gone after a packet has been re-transmitted the sender will re-transmit the packet with a timer twice the previous value (up to a threshold timer limit).

Error Detection - Checksums

Checksums in TCP work by processing a word (16 bytes) at a time and taking the sum of all words using one's complement arithmetic. The sum is then bitwise complemented and the checksum is inserted into the data header. This checksum is weak by modern standards as a error-correction method; however, it is still highly likely to identify a broken packet due to a checksum mismatch between the reciever and sender.
